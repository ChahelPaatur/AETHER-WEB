Before you start

Requirements.

Four things to have on hand. Everything else AETHER discovers for you.

Req

Python 3.9+

Any modern CPython interpreter on Linux, macOS, or Raspberry Pi OS.

Req

A robot kit.

Any supported kit — or any kit that ships a Python library.

Req

Anthropic API key.

Inference runs through the Claude API; the device only orchestrates.

Optional

Raspberry Pi 4 / 5.

Optional. Runs comfortably on a Pi 4 — no GPU required.

Step 01

Install.

$ pip install aether-robotics

On Raspberry Pi OS Bookworm:

$ pip install aether-robotics --break-system-packages
$ export PATH="$HOME/.local/bin:$PATH"
Step 02

Configure.

$ export ANTHROPIC_API_KEY="your-key-here"

For persistence, add the line to ~/.bashrc.

Step 03

Run.

$ aether --mode agent

You'll see auto-discovery scroll past, then an objective> prompt. Type plain English:

objective> look up
objective> look down
objective> drive forward and stop 10 centimeters before the wall
⌁

Ctrl-C fires the full emergency-stop chain — try it mid-skill. Every path leads to motors stopped and pins released.

End to end

Quick start — Raspberry Pi + PiCar-X.

The full sequence from a fresh OS to running AETHER:

# Enable I2C, SPI, Serial (interactive)
$ sudo raspi-config

# System packages
$ sudo apt update && sudo apt install -y python3-pip git i2c-tools

# SunFounder libraries
$ cd ~
$ git clone https://github.com/sunfounder/robot-hat.git
$ git clone https://github.com/sunfounder/picar-x.git
$ cd robot-hat && sudo python3 install.py
$ cd ../picar-x && sudo python3 setup.py install

# Permissions (prevents sudo prompts at runtime)
$ sudo mkdir -p /opt/picar-x /opt/robot-hat
$ sudo chown -R pi:pi /opt/picar-x /opt/robot-hat

# AETHER
$ pip install aether-robotics --break-system-packages
$ export PATH="$HOME/.local/bin:$PATH"
$ export ANTHROPIC_API_KEY="your-key"

# Run
$ aether --mode agent

You'll see [ADAPTER] Tier 1: sunfounder_picarx — 12 tools registered and [SKILLS] Registered 1 parallel skill(s): drive_until_obstacle in the boot log. Then go.

Programmatic

Python API.

from aether.core.tool_registry import get_global_registry
from aether.skills.drive_until_obstacle import drive_until_obstacle

registry = get_global_registry()

# Direct tool call
registry.execute("move_forward", {"speed": 30, "duration": 2.0})

# Skill call
result = drive_until_obstacle(target_cm=10, max_speed=25, timeout_s=15)
Over the wire

HTTP API.

$ aether --mode server  # starts on port 8000
$ curl -X POST http://localhost:8000/objective \
  -H "Content-Type: application/json" \
  -d '{"objective": "drive forward and stop 10 cm before the wall"}'
When things go sideways

Troubleshooting.

SymptomFix
aether: command not foundexport PATH="$HOME/.local/bin:$PATH"
Ultrasonic returns -1Wiring. Check TRIG=D2, ECHO=D3, power red=3V3/5V, black=GND.
GPIO busyAnother process holds pins. Reboot, or sudo pkill -f python.
[sudo] password for pi: during bootOld AETHER. Upgrade to 3.7.10+.
Sensor noise during drivingSolved in v3.9.5 (consecutive bad-read counter tolerates motor EMI).
